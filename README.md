# 🚖 OLA Ride Data Analysis | Data Analyst Project

## 📌 Overview
This project is an end-to-end data analysis case study based on OLA ride booking data in Bengaluru. It covers the complete workflow of a data analyst — from data generation and cleaning to querying and visualization.

The dataset contains 100,000+ rows of simulated ride data with realistic conditions like booking success rate, cancellations, and ride trends.

---

## 🧰 Tech Stack
- Google Sheets – Data generation & cleaning  
- SQL Workbench – Data querying & analysis  
- Power BI – Dashboard creation & visualization  

---

## 📊 Dataset Description
- City: Bengaluru  
- Duration: 1 Month  
- Records: 100K+  

### Key Columns:
- Booking ID, Date, Time  
- Customer ID  
- Vehicle Type (Auto, Sedan, SUV, Bike, etc.)  
- Booking Status  
- Pickup & Drop Location  
- Ride Distance & Booking Value  
- Cancellation Reasons  
- Customer & Driver Ratings  

---

## 🔍 Problem Statements

### SQL Analysis
- Retrieve all successful bookings  
- Find average ride distance per vehicle type  
- Count total cancellations (customer & driver)  
- Identify top 5 customers by number of rides  
- Analyze driver ratings (min & max)  
- Calculate total revenue from successful rides  
- Extract incomplete rides with reasons  

---

### Power BI Dashboard
- Ride volume over time  
- Booking status breakdown  
- Top vehicle types by ride distance  
- Revenue by payment method  
- Cancellation reasons analysis  
- Customer vs Driver ratings comparison  

---

## 📈 Key Insights
- ~62% of bookings were successfully completed  
- Customer cancellations were below 7%  
- Driver cancellations were below 18%  
- Higher booking values observed on weekends  
- Ratings analysis helped evaluate service quality
